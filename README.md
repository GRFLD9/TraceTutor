# TraceTutor

## Идея проекта

**TraceTutor** - это интерактивный тренажер-визуализатор выполнения Python-кода.

Пользователь загружает или вставляет небольшой фрагмент Python-кода, после чего программа пошагово показывает процесс его выполнения:
- какая строка исполняется в текущий момент;
- какой стек вызовов получился;
- какие локальные переменные существуют на каждом шаге;
- как меняется состояние программы;
- где и почему возникает исключение.

Проект ориентирован на обучение и разбор сложных механизмов Python. Его цель - сделать выполнение кода наглядным, понятным и удобным для изучения.

## Функциональность по итерациям

### Итерация 1
1. Контролируемый запуск и пошаговая трассировка Python-кода.
2. Отображение текущей строки, стека вызовов и локальных переменных.
3. Навигация по trace и отображение исключений.

### Итерация 2
1. Визуализация объектов и изменений состояния программы.
2. Учебный режим для разбора отдельных сценариев выполнения кода.
3. Текстовые объяснения, тесты и профилирование.

## Предварительная архитектура проекта

Планируется выделить следующие основные модули:

- `runner` - запуск небольших учебных Python-примеров для трассировки;
- `tracer` - сбор событий выполнения программы;
- `state` - хранение состояний шагов, фреймов и переменных;
- `renderer` - отображение состояния выполнения в интерфейсе;
- `explanations` - текстовые объяснения и учебный режим;
- `exceptions` - собственные исключения проекта;
- `tests` - набор тестов для основных компонентов.

## Пример сценария использования

1. Пользователь вставляет Python-код в приложение.
2. Программа запускает его в контролируемой среде.
3. На каждом шаге пользователь видит:
   - текущую исполняемую строку;
   - стек вызовов;
   - локальные переменные;
   - изменения состояния программы.
4. После завершения выполнения пользователь может просмотреть весь trace и разбор исключений.

## Основные сущности итерации 1

В первой итерации в проекте выделены несколько основных сущностей.

### `CodeRunner`

Класс из модуля `runner`.

Отвечает за контролируемый запуск пользовательского Python-кода:

- проверяет, что код не пустой;
- компилирует код через `compile`;
- запускает код через `exec`;
- включает трассировку через `sys.settrace`;
- перенаправляет `stdout` и `stderr`;
- возвращает итоговый объект `TraceResult`;
- фиксирует синтаксические ошибки и runtime-исключения.

### `ExecutionTracer`

Класс из модуля `tracer`.

Отвечает за сбор шагов выполнения программы. Использует callback-механизм `sys.settrace`, который вызывается интерпретатором Python на событиях:

- `call` - вызов функции;
- `line` - выполнение строки;
- `return` - возврат из функции;
- `exception` - возникновение исключения.

На основе этих событий `ExecutionTracer` создаёт последовательность объектов `TraceStep`.

### `TraceResult`

Dataclass из модуля `state`.

Хранит полный результат одного запуска пользовательского кода:

- исходные строки программы;
- список шагов выполнения;
- полный `stdout`;
- полный `stderr`;
- информацию об исключении, если оно произошло.

Также содержит свойство `is_success`, которое показывает, завершился ли запуск без исключения.

### `TraceStep`

Dataclass из модуля `state`.

Описывает один шаг выполнения программы:

- номер шага;
- тип события;
- номер текущей строки;
- имя текущей функции;
- текст текущей строки;
- стек вызовов;
- текущее содержимое `stdout`;
- возвращаемое значение для события `return`;
- информацию об исключении для события `exception`.

### `FrameSnapshot`

Dataclass из модуля `state`.

Описывает один фрейм в стеке вызовов:

- имя функции;
- номер строки;
- локальные переменные, существующие в этом фрейме.

### `VariableSnapshot`

Dataclass из модуля `state`.

Описывает одну локальную переменную:

- имя переменной;
- строковое представление значения;
- имя типа значения.

Значения переменных сохраняются через безопасное ограниченное `repr`, чтобы интерфейс не ломался из-за слишком длинного или некорректного вывода.

### `ExceptionSnapshot`

Dataclass из модуля `state`.

Хранит информацию об ошибке:

- тип исключения;
- сообщение исключения;
- номер строки, если его удалось определить.

Используется как для синтаксических ошибок, так и для ошибок времени выполнения.

### `EventKind`

Enum из модуля `state`.

Описывает поддерживаемые типы событий трассировки:

- `CALL`;
- `LINE`;
- `RETURN`;
- `EXCEPTION`.

### `StepExplainer`

Класс из модуля `explanations`.

Формирует короткие текстовые пояснения к шагам выполнения. Используется для учебного режима и подготовки к более подробным объяснениям во второй итерации.

### `TraceTutorApp`

Класс из модуля `renderer`.

Интерактивное TUI-приложение на базе библиотеки `textual`.

Отвечает только за отображение уже собранного trace:

- показывает исходный код;
- отображает текущий шаг;
- показывает стек вызовов;
- показывает локальные переменные;
- отображает `stdout`;
- показывает информацию об исключении;
- обрабатывает навигацию по шагам.

### Собственные исключения проекта

В модуле `exceptions` определены собственные исключения:

- `TraceTutorError` - базовый класс ошибок проекта;
- `EmptyCodeError` - ошибка пустого фрагмента кода;
- `StepLimitExceeded` - остановка выполнения при превышении лимита шагов;
- `UnsafeImportError` - ошибка импорта модуля вне разрешённого списка.

Эти исключения отделяют ошибки TraceTutor от обычных ошибок пользовательского кода.

## Установка и запуск

Создание виртуального окружения:

```bash
python3 -m venv .venv
```

Активация виртуального окружения:

```bash
source .venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
pip install -e .
```

Запуск приложения:

```bash
python -m tracetutor
```

Запуск тестов:

```bash
pytest
```

## Ограничения проекта

TraceTutor предназначен для запуска небольших учебных Python-примеров и визуализации их выполнения.

Проект не является sandbox-средой и не предназначен для безопасного запуска произвольного недоверенного кода. Предполагается, что пользователь запускает собственные учебные фрагменты кода.
